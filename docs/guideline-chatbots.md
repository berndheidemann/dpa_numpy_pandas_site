# Guideline: Chatbots in der Lernsituation nutzen

## 1. Grundhaltung
- Ziel ist das Lernen, nicht das schnelle Ergebnis. Ein Chatbot ist ein Coach und kein Ersatz für eigenes Denken.
- Verstehen vor Verwenden: Übernimm Code nur, wenn du ihn nachvollziehen kannst.

## 2. Umgang mit Antworten
- Nutze ausschließlich den Chat, nicht den autonomen Agenten – die Verantwortung bleibt bei dir.
- Prüfe Antworten kritisch: Passen sie zu deinem Projekt und den eingesetzten Technologien?
- Code abtippen statt kopieren:
  - unterstützt das Lernen
  - fördert ein tieferes Verständnis für Syntax
  - produziert Tippfehler, wodurch du lernst zu debuggen
  - zwingt dich, bewusst zu entscheiden, was du wirklich brauchst

## 3. Lernstrategien
- Formuliere Fragen wie eine Entwicklerin oder ein Entwickler, z. B.:
  - "Wie kann ich mit React Router navigieren?"
  - "Wie sieht ein Minimalbeispiel für einen Context aus?"
  - "Warum sollte ich hier einen `useEffect` statt eines Services verwenden?"
- Der Chat soll Neugier wecken und keine schnellen Lösungen liefern.
- Frage nach Prinzipien statt nach reinen Lösungen, z. B. "Warum braucht man Debouncing?".
- Bewerte Vorschläge kritisch: Nicht alles ist Best Practice, manches ist veraltet oder zu komplex.

## 4. Code-Ethik
- Passe Vorschläge immer an – verwende Code nur, wenn du weißt, warum er an dieser Stelle nötig ist.
- Erarbeite dir Fehler: Debugging ist ein Lernwerkzeug und kein Grund, sofort nach neuer Hilfe zu fragen.
- Dokumentiere bewusst: Kurze Kommentare erleichtern das Nachvollziehen.

## 5. Zusammenarbeit im Team
- Erkläre deine Lösung einer anderen Person, um dein Verständnis zu prüfen.
- Diskutiere statt Copy-Paste: Entscheidet gemeinsam, ob ein Ansatz passt.
- Führt einen gemeinsamen Qualitätscheck durch, damit der Code konsistent bleibt.
